# 개발 실행 계획 (Pre-Development to MVP)

버전: v1.0  
작성일: 2026-03-06

## 1. 단계별 계획

### Phase 0: 문서 기반 정합성 확정 (현재)
- 요구사항/기능/시나리오/테스트/정합성 리뷰 확정
- 스키마/권한/아키텍처/설치 가이드 확정
- 고위험 이슈(탈퇴 정책, 공개 범위) 의사결정

### Phase 1: 프로젝트 기본 골격
- Next.js 앱 생성 및 라우팅 구조 정렬
- Supabase 클라이언트/미들웨어/환경변수 구성
- 인증 화면과 기본 라우트 보호 구현

### Phase 2: 핵심 데이터 연동
- DB 마이그레이션 실행
- 레시피 CRUD API/Server Action 구현
- RLS와 동작 검증(개인 레시피 분리)

### Phase 3: UI/UX 완성
- 목록/작성/수정/상세 페이지 구현
- 검색/정렬/필터링/빈 상태 처리
- 이미지 업로드/미리보기 연결

### Phase 4: 공개/공유/보안 강화
- 공개 링크 공유 동작
- 정책에 따른 권한 분기 UI
- 에러 처리, toast, 접근성 보강

### Phase 5: QA 및 런치 준비
- 테스트케이스 기반 검증
- 보안 체크, 성능 초간단 측정
- 배포 파이프라인 및 환경 분리

## 3. 정의된 완료 기준(DoD)
- 모든 핵심 API가 문서 요구사항 및 보안 정책과 일치
- RLS에서 소유자 외 접근이 차단됨
- TC-A1~TC-S5 필수 경로 통과
- 민감 키 노출 없이 배포 가능한 환경설정 완성
