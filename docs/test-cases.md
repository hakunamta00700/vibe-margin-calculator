# 레시피 저장형 서비스 테스트케이스

버전: v1.0  
작성일: 2026-03-06  

## 테스트 전제
- 스테이징 환경의 Supabase 프로젝트 사용
- 테스트 계정 A, B 생성됨
- 테스트 데이터는 테스트 환경에서만 사용

## 1) 인증
| ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
|---|---|---|---|---|
| TC-A1 | 회원가입/로그인 | 미가입 사용자 | 회원가입 → 로그인 | 대시보드 진입 |
| TC-A2 | 잘못된 로그인 | 가입 사용자 | 틀린 비밀번호 입력 | 인증 실패 메시지 |
| TC-A3 | 비밀번호 재설정 | 가입 사용자 | 재설정 요청/메일 기반 변경 | 새 비밀번호로 로그인 성공 |
| TC-A4 | 로그아웃 | 로그인 상태 | 로그아웃 버튼 클릭 | 세션 종료, public 페이지 이동 |
| TC-A5 | 세션 유지 | 로그인 상태 | 새로고침 후 목록 재조회 | 로그인 상태 유지 |

## 2) 레시피 CRUD
| ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
|---|---|---|---|---|
| TC-C1 | 레시피 생성 | 로그인 상태 | 필수 항목 입력 후 저장 | 목록에 즉시 반영 |
| TC-C2 | 필수값 검증 | 로그인 상태 | 제목 미입력 저장 시도 | 저장 실패 + 안내 |
| TC-C3 | 조회 | 본인 레시피 존재 | 목록, 상세 접속 | 본인 데이터 노출 |
| TC-C4 | 수정 | 본인 레시피 존재 | 일부 항목만 변경 저장 | 변경분 반영, `updated_at` 갱신 |
| TC-C5 | 삭제 | 본인 레시피 존재 | 삭제 후 확인 | 목록에서 제거 |

## 3) 보안/권한
| ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
|---|---|---|---|---|
| TC-S1 | 본인 데이터 접근 제어 | 레시피 A(사용자 A 소유) | 사용자 B 세션에서 조회 | 403 또는 빈 결과 |
| TC-S2 | 비공개 차단 | 레시피 비공개 설정 | 타인 계정으로 링크 접근 | 조회 불가 |
| TC-S3 | 공개 조회 허용 | 레시피 공개 설정 | 비회원으로 공개 링크 접근 | 조회 가능 |
| TC-S4 | API 직접 조작 방어 | 로그인 사용자(B) | 임의 user_id/필터 조건 변경 | 서버 단에서 거부 |
| TC-S5 | 토큰 노출 검사 | 세션 상태 | 브라우저 스토리지/로컬노출 점검 | 민감 토큰이 노출되지 않음 |
| TC-S6 | 회원 탈퇴 즉시 삭제 | 사용자 A가 레시피 소유 | 탈퇴 요청 후 동일 계정으로 재접속/조회 | 계정 데이터 즉시 삭제, 복구 불가 |

## 4) 검색/정렬/공유
| ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
|---|---|---|---|---|
| TC-U1 | 키워드 검색 | 다수 레시피 | 키워드 입력 | 관련 결과만 표시 |
| TC-U2 | 정렬 변경 | 다수 레시피 | 최신/수정/이름순 변경 | 요청 정렬대로 노출 |
| TC-U3 | 카테고리 필터 | 카테고리 존재 | 특정 카테고리 선택 | 해당 카테고리만 표시 |
| TC-U4 | 공유 링크 접근성 | 공개 레시피 | 링크 복사/접근 | 링크 직접 접근 가능 |
| TC-U5 | 비회원 공개 목록 조회 | 비로그인 | `/api/recipes/public` 또는 홈 접속 | 공개 데이터만 20개 제한 조회 |
| TC-U6 | 비회원 공개 검색 | 비로그인 | 검색어 입력 | 공개 데이터 내에서만 검색 결과 표시 |

## 5) 장애 처리
| ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
|---|---|---|---|---|
| TC-F1 | 이미지 업로드 실패 | 네트워크 불안정 | 업로드 중단 | 사용자 알림 + 재시도 노출 |
| TC-F2 | 저장 중 중복 클릭 | 느린 네트워크 | 저장 버튼 연속 클릭 | 중복 저장 최소화/단건 반영 |
| TC-F3 | 필드 스키마 불일치 | API 변경/응답 예외 | 잘못된 응답 처리 | 안전한 에러 처리 |

## 6) 수용 판정 기준
- TC-A1~TC-S5는 필수 통과 항목
- 각 핵심 경로는 최소 1회 이상 성공/실패 경로가 검증되어야 함
- 보안 항목은 실패 시 즉시 개발 이슈로 분류
