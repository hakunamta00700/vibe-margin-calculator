# 레시피 저장형 서비스 유저 시나리오

버전: v1.0  
작성일: 2026-03-06  

## 1) 기본 시나리오

### US-01: 회원가입 후 첫 레시피 저장
1. 사용자가 회원가입 후 로그인한다.
2. `/recipes/new`에서 레시피 정보를 입력한다.
3. 공개/비공개를 선택하고 저장한다.
4. `/recipes`에서 저장된 항목이 목록에 나타난다.

### US-02: 본인 레시피 수정
1. 사용자가 `/recipes`에서 레시피를 선택한다.
2. `/recipes/[id]/edit`에서 조리시간과 재료를 수정한다.
3. 저장 후 상세/목록에서 변경사항이 반영되고 `updated_at`이 갱신된다.

### US-03: 레시피 삭제
1. 사용자가 레시피 상세에서 삭제 버튼을 클릭한다.
2. 확인 다이얼로그에서 동의한다.
3. 목록에서 제거되고 안내 메시지가 표시된다.

### US-04: 공개 레시피 공유
1. 사용자가 레시피를 공개로 설정한다.
2. 상세 페이지 URL을 복사해 공유한다.
3. 비회원도 링크로 상세 내용을 조회한다.

### US-05: 검색/정렬
1. 목록에서 재료 키워드(예: “토마토”)를 입력한다.
2. 키워드 포함 레시피만 표시된다.
3. 사용자가 최근 수정순 정렬을 선택한다.
4. 최근 수정된 항목이 상위에 표시된다.

## 2) 경계 시나리오

### US-06: 비공개 레시피 무단 접근 차단
1. 사용자 A가 자신의 레시피를 비공개로 저장한다.
2. 사용자 B가 해당 레시피 ID로 접근한다.
3. 서버가 접근을 거부하고 조회 불가 메시지를 반환한다.

### US-07: 인증 만료 상태 처리
1. 사용자가 로그인 상태에서 작업 중 세션이 만료된다.
2. API 호출 시 인증 실패가 발생한다.
3. 로그인 페이지로 이동 후 재로그인 유도된다.

### US-08: 입력값 누락
1. 제목 또는 재료를 비워 둔 채 저장한다.
2. 프론트엔드에서 검증 실패 메시지가 먼저 뜨고 저장이 중단된다.
3. 저장 완료 전까지 데이터가 반영되지 않는다.

### US-09: 이미지 업로드 실패
1. 레시피 이미지 업로드 중 네트워크 오류 발생.
2. 실패 메시지를 표시하고 재시도 가능 상태가 된다.

### US-10: 로그아웃/재진입
1. 사용자가 로그아웃한다.
2. `/recipes` 접근 시 인증 필요 화면/메시지가 표시된다.

## 3) 운영 시나리오

### US-11: 회원 탈퇴 후 데이터 처리
1. 사용자가 계정 삭제를 요청한다.
2. 시스템이 보유 레시피 삭제/보관 정책을 안내한다.
3. 사용자 동의 후 정책에 따라 처리한다.

## 4) 수락/수행 기준
- 모든 시나리오에서 잘못된 권한 접근은 서버에서 거부되어야 하며, UI는 사용자 친화 메시지로 대체한다.
- 공개/비공개 전환은 즉시 반영되어야 한다.
