# 레시피 저장형 서비스 요구사항 정의서

버전: v1.1  
작성일: 2026-03-06  
대상: Next.js + React + Supabase Auth + Supabase DB

## 1. 제품 목표
사용자가 회원가입/로그인 후 본인만의 레시피를 저장하고, 조회·수정·삭제하며, 공개 레시피를 공유할 수 있는 개인 레시피 관리 서비스를 구축한다.

## 2. 핵심 가치
1. 레시피 보존: 개인의 지식/요리를 구조화해 영속 저장
2. 데이터 접근 통제: 본인 데이터 분리(최소 권한)
3. 재사용성: 검색·정렬로 자주 쓰는 레시피를 빠르게 탐색
4. 공개 확장: 공개 레시피를 비회원에게도 최소 범위로 노출

## 3. 대상 사용자
- 일반 사용자: 개인 레시피를 저장·관리
- 운영자: 보안/정책 위반 점검

## 4. 범위(Scope)
### In Scope
- 회원가입/로그인/로그아웃/비밀번호 재설정
- 레시피 CRUD
- 공개/비공개 제어
- 검색/정렬/카테고리 필터
- 공개 레시피 목록/상세 공개
- 이미지 업로드 및 대표 이미지 교체

### Out of Scope (초기)
- 실시간 공동 편집
- 결제/구독
- 개인화 추천 엔진(고도화)

## 5. 기능적 요구사항 (FR)
### 인증
- FR-01: 사용자는 회원가입/로그인을 할 수 있다.
- FR-02: 사용자는 비밀번호 재설정을 요청할 수 있다.
- FR-03: 사용자는 로그아웃할 수 있다.
- FR-04: 인증 세션은 새로고침 후에도 유지된다.

### 레시피 관리
- FR-05: 사용자는 레시피를 생성할 수 있다.
- FR-06: 사용자는 본인 레시피를 목록/상세로 조회할 수 있다.
- FR-07: 사용자는 본인 레시피를 수정할 수 있다.
- FR-08: 사용자는 본인 레시피를 삭제할 수 있다.
- FR-09: 레시피는 제목, 설명, 재료, 조리 단계, 조리시간, 분량, 카테고리, 공개 여부를 가진다.
- FR-10: 공개/비공개 설정을 변경할 수 있다.
- FR-11: 사용자는 목록을 검색/정렬/카테고리 필터링할 수 있다.
- FR-12: 사용자는 공개 레시피 상세를 조회할 수 있다.
- FR-13: 사용자는 공개 레시피 목록을 제한 범위로 조회할 수 있다.

### 공개 범위(확정)
- FR-14: 비회원은 다음 공개 데이터만 접근 가능하다.
  - `is_public=true` 레시피 상세 페이지
  - 공개 레시피 목록(기본 정렬: 최신순, 제한: 20개)
  - 공개 레시피 검색(키워드 기반, 공개 데이터만)
- FR-15: 비회원은 비공개 데이터에 접근할 수 없다.

### 보안·안정성
- FR-16: 비회원이 비공개 레시피의 ID를 직접 조회할 수 없다.
- FR-17: 사용자에게는 입력값 검증 실패/권한 실패 메시지가 명시적으로 표시되어야 한다.
- FR-18: 이미지 업로드 실패 시 재시도 흐름이 제공되어야 한다.
- FR-19: 회원 탈퇴 시 본인 레시피는 즉시 삭제되어야 한다.

## 6. 비기능적 요구사항 (NFR)
- NFR-01: Supabase RLS가 사용자 단위로 데이터 분리를 강제한다.
- NFR-02: 권한 실패, 유효성 실패, 네트워크 실패에 대한 사용자 메시지를 일관되게 제공한다.
- NFR-03: 입력값은 클라이언트/서버 양측에서 검증한다.
- NFR-04: 모바일/데스크톱 모두에서 핵심 기능이 동작해야 한다.
- NFR-05: 접근성 기본 요건을 준수한다(라벨, 포커스, 대비).
- NFR-06: 인증 토큰은 노출되지 않으며 서버 우선 검증을 기본으로 둔다.
- NFR-07: 핵심 목록·상세 응답은 체감 2초 이내 반응을 목표로 한다.

## 7. 데이터 객체
- `users`: Supabase Auth(`auth.users`) 사용
- `recipes`: 레시피 본문 데이터
- `recipe_tags`: 태그(선택)
- `recipe_favorites`: 스크랩(선택)

## 8. 보안 정책
- 소유자 본인만 `recipes`의 쓰기/수정/삭제 가능
- 공개 조회는 `is_public=true`에서만 허용
- 비회원 공개 접근은 `features.md`의 공개 범위 규칙 준수
- Storage 정책은 업로더 소유자 우선으로 제한

## 9. 수용 기준
- 비로그인 접근은 개인 레시피 조회에 실패해야 한다.
- 로그인 사용자는 본인 레시피만 수정/삭제할 수 있어야 한다.
- 비공개 레시피는 타인 및 비회원에게 노출되지 않아야 한다.
- 회원 탈퇴 시 레시피 데이터는 즉시 삭제되어야 한다.

## 10. 운영 정책
- 이미지 용량 제한: `5MB`, 포맷: `jpg/png/webp`
- 탈퇴 정책: 계정 삭제 시 연관 레시피 및 저장 메타 즉시 영구 삭제
